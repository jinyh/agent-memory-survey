# 关键 Agent Memory 系统与框架

## 1. 概述

本文档详细介绍 Agent Memory 领域最具代表性的系统和框架，分析其架构设计、核心创新和适用场景。

## 2. 学术系统

### 2.1 A-MEM — Agentic Memory (NeurIPS'25)

- **论文**: arXiv:2502.12110
- **代码**: https://github.com/agiresearch/A-mem
- **核心思想**: 基于 Zettelkasten（卡片笔记法），LLM 自主构建互联记忆网络
- **架构**:
  - 每条记忆是一个"笔记"，包含内容、标签、链接
  - 新记忆整合时，LLM 评估与现有笔记的关联，创建双向链接
  - 动态索引：检索时综合内容相似度和链接图的拓扑距离
- **创新**: 记忆不是孤立条目，而是形成知识网络；新信息可触发历史记忆的上下文更新
- **结果**: 在六个基础模型上超越 SOTA 基线

### 2.2 AgeMem — Agentic Memory Management (2026)

- **论文**: arXiv:2601.01885
- **核心思想**: 将记忆操作暴露为 Agent 可调用的工具动作
- **架构**:
  - 记忆操作集: {Store, Retrieve, Update, Summarize, Discard}
  - 三阶段渐进式 GRPO 训练:
    1. 基础记忆读写
    2. 记忆管理策略（何时存/删）
    3. 长程任务中的自适应记忆使用
- **创新**: Agent 通过 RL 学习最优记忆管理策略，而非依赖预定义规则
- **结果**: 在五个长程基准上持续优于强基线

### 2.3 MemGPT → Letta

- **论文**: arXiv:2310.08560
- **官网**: https://docs.letta.com
- **核心思想**: 将 LLM 类比为操作系统，记忆管理类似虚拟内存
- **架构**:
  - **主上下文 (Main Context)**: 类似 RAM，当前对话的核心信息
  - **外部上下文 (Archival/Recall)**: 类似硬盘，持久化存储
  - Agent 通过函数调用自主在两层之间搬运数据
  - 支持跨会话持久化
- **创新**: 首次提出 OS 式记忆管理范式；Agent 拥有记忆的"写权限"
- **演化**: 2024年9月纳入 Letta 开源框架，成为完整 Agent 开发平台

### 2.4 Memory³ (Memory Cubed)

- **论文**: arXiv:2407.01178 (JML, 2024)
- **核心思想**: LLM 有三种记忆：隐式记忆（权重）、工作记忆（KV cache）、显式记忆（外化知识块）
- **架构**:
  - 显式记忆以压缩表示存储，可在推理时按需加载
  - 记忆电路理论：分析知识如何在三种记忆形式之间流动
  - 记忆稀疏化：减少冗余存储
- **创新**: 2.4B 参数模型通过显式记忆超越更大模型和 RAG 方案
- **结果**: 更高解码速度 + 更好性能

### 2.5 MemoRAG

- **论文**: arXiv:2409.05591 (TheWebConf'25)
- **代码**: https://github.com/qhjqhj00/MemoRAG
- **核心思想**: 双系统架构，全局记忆指导精确检索
- **架构**:
  - **系统 1 (快系统)**: 轻量模型创建全局记忆（KV 压缩），生成草稿答案作为检索线索
  - **系统 2 (慢系统)**: 高表达力模型基于检索结果生成精确答案
  - RLGF (Reinforcement Learning from Generation Feedback) 强化记忆能力
- **创新**: 将全局理解与精确检索解耦，解决"模糊查询"难题
- **结果**: 在非 QA 任务上优势显著，超越 HyDE, RQ-RAG, GraphRAG

### 2.6 MemWalker

- **论文**: arXiv:2310.05029
- **核心思想**: LLM 在记忆树上交互式导航
- **架构**:
  - 离线构建查询无关的记忆树（叶节点=文本块，内部节点=摘要）
  - 查询时从根节点开始，逐层选择最相关的子节点
  - 支持回溯：15-20% 导航触发 rollback，其中 60-80% 最终成功
- **创新**: 将记忆检索从一次性查询变为交互式探索

### 2.7 MSA — Memory Sparse Attention

- **论文**: ref/MSA__*.pdf
- **核心思想**: 端到端可训练的稀疏注意力，支持 100M token 记忆
- **架构**:
  - Router Projector 生成路由键，top-k 文档选择
  - Document-wise RoPE 解耦文档间位置编码
  - 三阶段推理: 离线编码 → 路由装配 → 稀疏生成
  - Memory Interleave: 迭代式多跳推理
  - 分层存储: GPU 路由键 + CPU 内容 KV
- **创新**: 首次在隐状态范式中同时达到终生记忆 + 高精度 + 端到端可训练
- **结果**: 16K→100M tokens 仅 <9% 性能退化

### 2.8 Zep/Graphiti

- **论文**: arXiv:2501.13956
- **核心思想**: 时序知识图谱作为 Agent 记忆后端
- **架构**:
  - 知识图谱节点=实体/概念，边=关系，带时间有效期
  - 三种搜索: 全文搜索（词汇）、余弦相似度（语义）、BFS（图上下文）
  - 非有损动态更新：新事实不覆盖旧事实，维护时间线
- **创新**: 解决知识图谱的时间维度问题，事实有"有效期"
- **结果**: DMR 基准 94.8%，优于 MemGPT 93.4%

## 3. 工程框架

### 3.1 框架对比

| 框架 | 记忆类型 | 持久化 | 多 Agent | 特点 |
|------|---------|--------|---------|------|
| **Letta** | 分层（主+外部）| 是 | 否 | OS 式管理，自编辑 |
| **LangChain** | 模块化（Buffer/Summary/Entity）| 可选 | 否 | 可插拔设计，生态最大 |
| **LlamaIndex** | RAG-first | 是 | 否 | 擅长大规模文档上下文 |
| **CrewAI** | 短期+用户记忆 | Mem0 集成 | 是 | 多 Agent 共享 |
| **Mem0.ai** | 独立记忆层 | 是 | 间接 | 可集成多框架 |
| **Zep** | 时序知识图谱 | 是 | 否 | 自动摘要，会话级管理 |

### 3.2 选型建议

- **快速原型/POC**: LangChain Memory（Buffer/Summary 即可）
- **长期 Agent 应用**: Letta（持久化 + 自编辑）或 Zep（结构化记忆）
- **多 Agent 协作**: CrewAI + Mem0
- **大规模文档检索**: LlamaIndex + MemoRAG 思路
- **研究/自定义**: 基于本项目原型代码扩展
