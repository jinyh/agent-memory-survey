# 研究工作流

> v1.0.0 | 2026-03-26

本项目的研究主线固定为：

`问题定义 -> 证据建模 -> 研究判断 -> 架构决策 -> 原型计划 -> 实验评测 -> survey/报告回写`

这条链路的目的不是增加流程感，而是确保每个结论都能回答三个问题：

1. 这个判断来自什么证据。
2. 这个判断落成了什么设计或实验。
3. 这个实验结果最终改变了什么研究结论。

## 阶段 1：问题定义

目标：明确研究问题、范围、非目标和成功标准。

默认产物：

- `research-brief`

进入条件：

- 有明确研究主题，但尚未锁定证据和实验边界

退出条件：

- 问题、范围、非目标、成功标准已写清
- 已列出首批证据入口

## 阶段 2：证据建模

目标：把 `paper / blog / DeepResearch` 分层整理，形成可追溯的证据图谱。

默认产物：

- `evidence-map`
- 必要时补充 `docs/references/` 索引

进入条件：

- 已有 `research-brief`

退出条件：

- 代表引用明确
- 主证据与补充证据边界明确
- 争议点和证据缺口被标注

## 阶段 3：研究判断

目标：把证据转成结构化判断，而不是停留在资料堆积。

默认产物：

- survey 章节草稿或研究判断摘要

进入条件：

- 已完成 `evidence-map`

退出条件：

- 关键判断与代表引用绑定
- 明确哪些是论文证据，哪些是工程判断，哪些是综合推断

## 阶段 4：架构决策

目标：将研究判断落为原型边界、接口约束和实验前提。

默认产物：

- `architecture-decision`

进入条件：

- 有明确研究判断需要落地到 `src/`、工具链或评测逻辑

退出条件：

- 决策、替代方案、影响面、风险已记录
- 通过 Gate A

## 阶段 5：原型计划

目标：把架构决策变成可执行的实验或实现计划。

默认产物：

- `experiment-spec`
- 必要时新增实施计划

进入条件：

- 已有 `architecture-decision`

退出条件：

- 指标、输入输出、ground truth、失败条件明确
- 通过 Gate B

## 阶段 6：实验评测

目标：执行实现与评测，形成可复现的结果与边界说明。

默认产物：

- `evaluation-report`
- 代码、测试、评测产物

进入条件：

- 已有 `experiment-spec`

退出条件：

- 指标结果、案例、trace、失败样例齐备
- 结果能映射回假设或决策

## 阶段 7：survey/报告回写

目标：将实验与设计结论回写到研究叙述层，而不是让结果停留在实现侧。

默认产物：

- `survey-update-note`
- 更新后的 `docs/survey/` 章节或其他研究文档

进入条件：

- 已有 `evaluation-report`

退出条件：

- 结论边界写清
- 通过 Gate C
- `survey` 文本能追到实验与证据来源
