# 关键评审门

> v1.0.0 | 2026-03-26

本项目不追求全流程强制，只在关键节点设置评审门。

## Gate A：研究判断进入实现前

目的：

- 防止研究结论未经架构决策就直接落成代码

必须具备：

- `research-brief`
- `evidence-map`
- `architecture-decision`

审查重点：

- 研究判断是否有证据支撑
- 设计是否明确了替代方案与影响面
- 是否存在“用实现反推结论”的风险

默认责任人：

- `Architect`
- `Reviewer`

未通过时回退：

- 回到证据建模或架构决策阶段

## Gate B：实验开始前

目的：

- 防止在没有明确指标、ground truth 或失败条件时启动实验

必须具备：

- `experiment-spec`
- 对应 `decision_id`

审查重点：

- 指标是否能回答研究问题
- ground truth 或替代判定口径是否明确
- 失败条件是否可操作

默认责任人：

- `Experiment Engineer`
- `Reviewer`

未通过时回退：

- 回到原型计划阶段

## Gate C：结果进入 survey 前

目的：

- 防止把局部实验结果直接写成一般性研究结论

必须具备：

- `evaluation-report`
- `survey-update-note`

审查重点：

- 结果是否能追到实验与决策
- 是否区分论文证据、工程判断、综合推断
- 是否说明失败样例与边界条件

默认责任人：

- `Writer`
- `Reviewer`

未通过时回退：

- 回到实验评测或研究判断阶段

## 例外规则

以下情况允许轻量跳过部分流程，但必须在文档中显式说明：

- 仅修正文案、不改变研究判断
- 仅修复实现缺陷、不改变实验口径
- 仅补充已有结论的代表引用，不新增组织性判断

即使走例外，也不能跳过：

- 证据层级约束
- survey 中的边界说明
- 关键 ID 追踪信息
