# survey-evidence-mapper

项目级 skill，用于把新论文快速映射到 AgentResearch 仓库的证据结构中，而不是做泛泛的论文总结。

## 适用场景

当你想让 Claude 回答下面这类问题时，应该触发这个 skill：

- 这篇论文值不值得纳入当前仓库？
- 它最适合挂到哪个 `docs/survey/*.md` 章节？
- 它应该算主证据、补充证据，还是工程说明？
- 它支撑什么，不支撑什么？
- 是否应该更新 `docs/ideas`、`docs/survey`、`ref/paper` 或 `docs/survey/survey-map.md`？

## 默认工作方式

这个 skill 默认只做**分析**，不改文件。

默认输出四部分：

1. `快速判断`
2. `可借鉴点`
3. `边界`
4. `建议动作`

只有当用户明确要求时，才进入**分析 + 最小落笔**模式，去更新仓库中的文档。

## 项目绑定的证据层

它围绕这个仓库的四层结构工作：

- `ref/paper/`：论文库存放层
- `docs/ideas/`：中间判断与启发记录层
- `docs/survey/*.md`：正式 survey 论证层
- `docs/survey/survey-map.md`：索引层，不替代正文

## 使用边界

这个 skill 不适合：

- 通用论文综述
- 全文翻译
- 论文复现
- 与当前仓库无关的研究调研

## 示例提示词

- `帮我看 ref/paper/2603.03596.pdf 对这个仓库有没有借鉴价值，不要泛泛总结，只说支撑什么、不支撑什么，以及最适合挂到哪个章节。`
- `这篇 paper 应该算 07-frontiers 的主证据还是补充证据？`
- `先判断这篇论文的正确落点；如果结论明确，再最小化更新相关 survey 文档。`
